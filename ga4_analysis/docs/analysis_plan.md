# 分析計画書｜工務店GA4分析プロジェクト

作成日：2026-05-27  
対象：工務店・住宅会社のWebサイト

---

## 1. 分析の目的と背景

工務店・住宅会社のWebサイトでは、**訪問者が多くても問い合わせにつながらない**という課題が多い。  
GA4のデータを活用し、「問い合わせしたユーザー」と「しなかったユーザー」の行動差を明らかにすることで、  
**具体的な改善箇所と施策の優先順位**を提示することを目的とする。

---

## 2. コンバージョン（CV）の定義

このプロジェクトでは以下をコンバージョン（成果）として定義する。

| CV名称         | 内容                         | GA4イベント名（例）        |
|---------------|-----------------------------|-----------------------|
| 問い合わせ完了  | 問い合わせフォーム送信完了ページ表示 | `form_submit` / `page_view（/thanks）` |
| 資料請求完了   | 資料請求フォーム送信完了         | `resource_request`    |
| 来場予約完了   | 見学予約フォーム送信完了         | `reservation_complete` |
| 電話タップ     | スマホからの電話番号タップ        | `click（tel:）`         |

> **注意：** 実際に使用するGA4プロパティのイベント名は、分析開始前に確認・更新すること。

---

## 3. 分析対象期間

| 区分         | 期間設定              | 目的                         |
|-------------|----------------------|------------------------------|
| 基本分析期間  | 直近3ヶ月             | 現状把握・改善仮説の立案       |
| 比較期間     | 前年同月比 or 前期比  | 施策効果の検証                |
| トレンド確認  | 直近12ヶ月            | 季節変動・長期的な傾向把握     |

## 4. 必要なCSVとGA4エクスポート方法

### ① ユーザー行動サマリー（`user_overview.csv`）

**取得場所：** GA4 > レポート > 概要  
**取得項目：** ユーザー数、セッション数、エンゲージメント率、コンバージョン数、CV率  
**ディメンション：** 日付

---

### ② CV済みユーザーデータ（`cv_segment.csv`）

**取得場所：** GA4 > 探索 > 自由形式  
**設定方法：**
1. セグメントで「コンバージョン済みユーザー」を作成
2. ディメンション：ページタイトル、流入元/メディア
3. 指標：セッション数、エンゲージメント率、平均セッション時間

---

### ③ ページ別閲覧データ（`page_performance.csv`）

**取得場所：** GA4 > レポート > エンゲージメント > ページとスクリーン  
**取得項目：** ページタイトル、表示回数、ユーザー数、平均滞在時間、離脱率

---

### ④ 流入経路データ（`traffic_source.csv`）

**取得場所：** GA4 > レポート > 集客 > トラフィック獲得  
**取得項目：** セッションのデフォルトチャネルグループ、セッション数、CV数、CV率

---

### ⑤ ランディングページデータ（`landing_page.csv`）

**取得場所：** GA4 > 探索 > 自由形式  
**ディメンション：** ランディングページ  
**指標：** セッション数、エンゲージメント率、CV数、CV率

---

### ⑥ デバイス・地域データ（`device_region.csv`）

**取得場所：** GA4 > レポート > ユーザー属性 > 技術詳細 / 地域  
**取得項目：** デバイスカテゴリ（スマホ/PC/タブレット）、地域、ユーザー数、CV数

---

### ⑦ イベントデータ（`events.csv`）

**取得場所：** GA4 > レポート > エンゲージメント > イベント  
**取得項目：** イベント名、イベント数、ユーザー数

---

### ⑧ ファネルデータ（`funnel.csv`）

**取得場所：** GA4 > 探索 > ファネルデータ探索  
**設定：** トップページ → 施工事例 → 問い合わせページ → 完了ページ  
**取得項目：** 各ステップのユーザー数、離脱率

---

## 5. 分析項目と仮説一覧

### A. CV済みユーザーの行動分析

| 分析項目                     | 確認する問い                         | 工務店向け仮説例                              |
|-----------------------------|------------------------------------|--------------------------------------------|
| 閲覧ページ数の分布            | 何ページ見た人が問い合わせしやすいか？  | 5ページ以上見た人のCV率が高い傾向がある        |
| 滞在時間の分布               | 何分サイトにいた人が問い合わせするか？  | 3分以上滞在したユーザーがCVしやすい           |
| CV前の必須通過ページ           | 問い合わせ前に必ず見るページはどこか？  | 「施工事例」「価格・費用」ページが必須通過ページ |
| 問い合わせまでのステップ数     | 何回訪問後に問い合わせしているか？     | リピーター（2回目以降）のCV率が高い           |

---

### B. CV有/無ユーザーの比較

| 比較軸                      | CV済みユーザー（想定）  | 非CVユーザー（想定）       |
|----------------------------|--------------------|------------------------|
| 平均閲覧ページ数             | 多い（5〜8ページ）    | 少ない（1〜2ページ）      |
| 平均滞在時間                 | 長い（3〜5分）       | 短い（1分未満）          |
| 施工事例ページの閲覧率        | 高い               | 低い                   |
| スマホ比率                   | やや低め            | 高め（スマホ最適化が課題）  |

---

### C. 流入経路別パフォーマンス

| チャネル            | 期待されるCV率  | 主な確認ポイント                      |
|--------------------|-------------|-------------------------------------|
| オーガニック検索     | 中〜高       | 検索キーワードと着地ページの一致度      |
| 有料検索（リスティング） | 高（コスト大） | LP別CV率、費用対効果                  |
| SNS（Meta/Instagram） | 低〜中      | エンゲージメント率、認知からの育成      |
| 直接流入            | 高           | ブランド認知度、リピーター率            |
| 紹介（外部リンク）   | 中           | 紹介元サイトの質                      |

---

### D. ページ改善の優先度マトリクス

```
         訪問数 高 ←————————→ 訪問数 低
              |                    |
CV率 高  | 【維持・強化】         | 【育成対象】    |
         |  ここは伸ばし続ける     | 認知を上げたい  |
         |------------------------+----------------|
CV率 低  | 【最優先改善】         | 【後回しでOK】  |
         |  訪問多いのに勿体ない！  | 優先度低め     |
              |                    |
```

## 6. 分析の進め方（ステップ順）

```
Step 1：データ確認（01_data_check.ipynb）
  → CSVの読み込み、欠損値・異常値の確認
  → CVイベント名の特定・整合確認

Step 2：CV済みユーザー分析（02_cv_behavior.ipynb）
  → 行動パターンの可視化
  → 「問い合わせにつながる行動」の仮説抽出

Step 3：CV有/無比較（03_comparison.ipynb）
  → 2グループの差異を数値・グラフで表現
  → 改善ポイントの特定

Step 4：流入経路分析（04_traffic_source.ipynb）
  → チャネル別CV率の比較
  → 広告・SEO・SNSの効果評価

Step 5：ページ分析（05_page_analysis.ipynb）
  → 離脱ページ・通過ページの特定
  → 改善優先度マトリクスの作成

Step 6：レポート出力
  → エグゼクティブサマリー作成
  → 改善アクションシート作成
```

---

## 7. レポート出力形式

### エグゼクティブサマリー（経営者向け）
- 今月の成果を3行で要約
- 改善した点 / 課題点を箇条書き
- 今すぐやるべき施策TOP3

### 詳細分析レポート（広報・担当者向け）
- 各分析テーマのグラフ＋コメント
- 平易な言葉で記述（専門用語は注釈付き）

### 改善アクションシート（実行管理用）
| 優先度 | 改善箇所 | 現状の課題 | 推奨アクション | 期待効果 |
|-------|---------|----------|-------------|---------|

## 8. 用語集（工務店向け言い換え表）

| GA4専門用語        | このレポートでの言い換え               |
|------------------|--------------------------------------|
| コンバージョン      | 問い合わせ・資料請求・来場予約          |
| セッション         | 訪問（回）                           |
| ユーザー           | 訪問者                              |
| エンゲージメント率  | ちゃんと見てくれた訪問の割合           |
| 直帰              | 1ページだけ見てすぐ離脱               |
| 離脱率            | そのページを最後に帰ってしまった割合    |
| オーガニック検索   | Google検索からの自然流入              |
| ランディングページ  | 最初に開いたページ                    |
| ファネル          | 問い合わせまでの道筋                  |
| インプレッション   | 広告が表示された回数                  |
| CTR              | 広告をクリックした割合                |
| CPA              | 問い合わせ1件あたりの広告費            |
